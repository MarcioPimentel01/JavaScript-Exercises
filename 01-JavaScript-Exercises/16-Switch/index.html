16 Switch

The switch statement in JavaScript provides a way to execute different blocks of code based on the value of an expression. It's an alternative to using multiple if statements when you need to compare the same value against multiple possible cases.

The switch statement and the if-else statement in JavaScript are both used for control flow, allowing you to execute different blocks of code based on conditions. However, they have different use cases and syntax:

Lets consider the if-else example below:

        
            let day; 

            let day; // The variable 'day' is declared but not initialized, so its value is undefined.

            if (day === 1) { // Check if 'day' is equal to 1.
                console.log(`Today's Monday`);
            } else if (day === 2) { // Check if 'day' is equal to 2.
                console.log(`Today's Tuesday`);
            } else if (day === 3) { // Check if 'day' is equal to 3.
                console.log(`Today's Wednesday`);
            } else if (day === 4) { // Check if 'day' is equal to 4.
                console.log(`Today's Thursday`);
            } else if (day === 5) { // Check if 'day' is equal to 5.
                console.log(`Today's Friday`);
            } else if (day === 6) { // Check if 'day' is equal to 6.
                console.log(`Today's Saturday`);
            } else if (day === 7) { // Check if 'day' is equal to 7.
                console.log(`Today's Sunday`);
            } else { // If 'day' doesn't match any of the conditions above, execute this block.
                console.log(`Invalid Day`);
            }

Switch, uses the following structure

            
                switch (expression) {
                    case value1:
                        // Code to execute if expression matches value1
                        break;
                    case value2:
                        // Code to execute if expression matches value2
                        break;
                    default:
                        // Code to execute if expression doesn't match any case
                }

Key differences:

  1. Expression evaluation:

    • switch: The expression is evaluated once, and the corresponding case is executed. It's more efficient when you have a large number of cases because it avoids repeated evaluations of the expression.
    • if-else: Conditions are evaluated from top to bottom until a true condition is found.

  2. Case Comparison:

    • switch: Cases are compared using strict equality (===). This means that both the value and the type of the expression must match the case value for the corresponding block to execute.
    • if-else: Conditions can be complex, involving any valid JavaScript expression.

  3. Fall-through behavior:

    • switch: Without a break statement, execution will "fall through" to the next case. This behavior can be used intentionally to execute multiple cases with the same code block.
    • if-else: Each condition is independent, and execution stops after the first true condition is found.

In summary, use switch when you have a single expression with multiple possible values to compare against, and use if-else when you need flexibility or have complex conditions.
