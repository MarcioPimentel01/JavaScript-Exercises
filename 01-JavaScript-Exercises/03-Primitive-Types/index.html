03 Primitive types

Primitive types in JavaScript refer to basic data types that are not objects and do not have methods. There are six primitive data types in JavaScript:

Number: Represents numeric values, including integers and floating-point numbers.

                    
                        let age = 30;
                        let height = 5.9;

String: Represents sequences of characters, enclosed in single (') or double (") quotes.

                    
                        let name = "John";
                        let message = 'Hello, world!';

Boolean: Represents a logical value, either true or false.

                    
                        let isStudent = true;
                        let hasCar = false;

Undefined: Represents a variable that has been declared but has not been assigned a value.

                    
                        let x;
                        console.log(x); // Output: undefined

Null: Represents the intentional absence of any value.

                      
                        let car = null;

Symbol: (added in ECMAScript 6): Represents a unique and immutable value used as an identifier for object properties. Symbols are often used as property keys when creating objects to avoid naming conflicts.

                    
                        const sym1 = Symbol();
                        const sym2 = Symbol("description");

Primitive types are immutable, meaning their values cannot be changed after they are created. When you manipulate a primitive value, you are working directly with the value itself, not a reference to it. This is in contrast to objects, where you work with references to the object in memory.

Understanding primitive types is essential in JavaScript because they form the basis for storing and manipulating data in your programs. Additionally, knowing the characteristics of each primitive type helps you write more efficient and effective code.
